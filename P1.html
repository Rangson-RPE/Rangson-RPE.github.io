หัวใจของการใช้ AI ไม่ใช่แค่การถาม แต่คือการให้ "บริบท" และ "โครงสร้าง" ที่ชัดเจน

The "RODES" Framework

ใช้โครงสร้างนี้เพื่อผลลัพธ์ที่แม่นยำที่สุดในทุกงาน

Role
กำหนดบทบาท AI
Objective
เป้าหมายที่ต้องการ
Details
รายละเอียด/ข้อจำกัด
Steps/Style
ขั้นตอนหรือรูปแบบผลลัพธ์
IT Management

เขียนนโยบายไอที (IT Policy)

สำหรับสร้างร่างนโยบายที่สอดคล้องกับมาตรฐาน

"คุณคือ IT Manager หน้าที่ของคุณคือร่างนโยบาย [ชื่อนโยบาย] โดยต้องอ้างอิงมาตรฐาน [ISO27001/PDPA] เน้นภาษาที่เป็นทางการและปฏิบัติได้จริง แบ่งหัวข้อเป็น: 1.วัตถุประสงค์ 2.ขอบเขต 3.ข้อบังคับ"
Data Analysis

วิเคราะห์ข้อมูล Solar / Energy

เน้นการหาความผิดปกติ (Anomaly Detection)

"ในฐานะ Data Analyst โปรดวิเคราะห์ข้อมูล [ชุดข้อมูล] เพื่อหาจุดที่ประสิทธิภาพลดลงผิดปกติ (Fouling) เปรียบเทียบระหว่างวันที่ [A] และ [B] สรุปเป็น 3 ประเด็นหลักพร้อมคำแนะนำ"
Python Coding

Debug และ Optimize Code

ลดการเขียนคำอธิบายที่ยืดเยื้อ เน้นที่โค้ด

"โปรดตรวจสอบโค้ด Python ต่อไปนี้ เพื่อหาจุดที่อาจเกิด Memory Leak และปรับปรุงให้ทำงานเร็วขึ้น แก้ไขโค้ดและอธิบายเฉพาะจุดที่สำคัญสั้นๆ เป็นข้อๆ"
Strategic Planning

วาง Roadmap AI 2026

สำหรับการนำเสนอผู้บริหาร (Executive Presentation)

"ทำหน้าที่เป็น Business Consultant ช่วยร่างแผนการนำ AI มาใช้ในองค์กรพลังงาน แบ่งเป็น 4 ไตรมาส โดยเน้นเรื่อง Quick Wins ในไตรมาสแรก และ Scalability ในช่วงท้าย สรุปเป็นตาราง"
